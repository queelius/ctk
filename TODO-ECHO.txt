Missing for Full Integration

1. MANIFEST.JSON

   longecho build uses manifest.json for configuration. Without it, longecho
   falls back to auto-discovery, but explicit manifests give better control.

   Add to export output:

   {
     "version": "1.0",
     "name": "Conversation Archive",
     "description": "AI conversation history",
     "type": "database",
     "browsable": true,
     "icon": "chat"
   }

   See: ~/github/beta/longecho/spec/MANIFEST-SCHEMA.md

2. SITE/ GENERATION (OPTIONAL)

   ctk already has HTML export. Consider generating site/ during ECHO export
   so longecho build can link to it directly instead of generating one.

   If site/ exists, longecho links to it. If not, longecho generates basic UI.
